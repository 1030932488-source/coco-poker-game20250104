# 最终完整操作步骤

## 一、环境准备（必须完成）

### 步骤1：安装必需软件

#### 1.1 安装 Node.js
1. 访问 https://nodejs.org/
2. 下载 LTS 版本（推荐 18.x）
3. 运行安装程序，**重要**：勾选 "Add to PATH"
4. 验证安装：
   ```bash
   node --version
   npm --version
   ```

#### 1.2 安装 Cocos Creator
1. 访问 https://www.cocos.com/creator
2. 下载最新版本（推荐 3.8.0 或更高）
3. 运行安装程序
4. 首次启动需要注册/登录账号

#### 1.3 安装 Git
1. 访问 https://git-scm.com/
2. 下载 Windows 版本
3. 运行安装程序
4. 配置 Git（首次使用）：
   ```bash
   git config --global user.name "你的名字"
   git config --global user.email "你的邮箱"
   ```

### 步骤2：验证环境

打开命令提示符（CMD）或 PowerShell，运行：

```bash
node --version    # 应显示版本号
npm --version     # 应显示版本号
git --version     # 应显示版本号
```

如果都显示版本号，说明环境配置成功。

## 二、项目配置（必须完成）

### 步骤1：打开项目

1. **启动 Cocos Creator**
   - 双击桌面图标
   - 等待启动完成

2. **打开项目**
   - 点击"打开项目"
   - 浏览到：`D:\coco_poker`
   - 点击"打开"

3. **等待编译**
   - Cocos Creator 会自动编译 TypeScript
   - 查看底部状态栏，等待编译完成
   - 如果有错误，检查 Node.js 是否正确安装

### 步骤2：配置项目设置

1. **打开项目设置**
   - 菜单：`项目` -> `项目设置`

2. **设置设计分辨率**
   - 宽度：`1080`
   - 高度：`2080`
   - 适配模式：`固定高度（FIXED_HEIGHT）`

3. **保存设置**
   - 点击"确定"保存

### 步骤3：创建场景

1. **创建场景文件**
   - 在资源管理器中，右键点击 `assets/scenes/`
   - 选择"新建" -> "场景"
   - 命名为 `GameScene`

2. **设置场景**
   - 选中场景根节点（Canvas）
   - 在属性检查器中：
     - 宽度：`1080`
     - 高度：`2080`

### 步骤4：创建节点结构

按照以下结构创建节点（详细步骤见"完整操作指南.md"）：

```
Canvas
├── MainCamera
├── PlayfieldArea (Node)
├── StackArea (Node)
│   ├── ReserveStack (Node)
│   └── BaseCardArea (Node)
└── UILayer (Node)
    ├── UndoButton (Button)
    └── ScoreLabel (Label) [可选]
```

### 步骤5：添加组件

1. **添加 GameScene 脚本**
   - 选中场景根节点
   - 添加组件：`GameScene`

2. **创建 GameController 节点**
   - 创建新节点 `GameController`
   - 添加组件：`GameController`

3. **配置组件引用**
   - 参考"完整操作指南.md"中的详细步骤
   - 确保所有引用都正确配置

## 三、运行测试（必须完成）

### 步骤1：运行预览

1. **点击预览按钮**
   - 点击编辑器顶部的"预览"按钮
   - 或按快捷键 `Ctrl + P`

2. **等待浏览器打开**
   - 游戏会在浏览器中运行

### 步骤2：测试功能

测试以下功能：

1. **卡牌匹配**
   - 点击主牌区的卡牌
   - 检查是否能与底牌匹配（点数差1）
   - 检查动画是否正常

2. **翻牌功能**
   - 当主牌区无可匹配的牌时
   - 点击备用牌堆
   - 检查是否能翻出新牌

3. **回退功能**
   - 执行几个操作
   - 点击回退按钮
   - 检查是否能正确回退
   - 测试连续多次回退

### 步骤3：修复问题

如果遇到问题：

1. **查看控制台**
   - 打开"控制台"面板
   - 查看错误信息

2. **检查组件引用**
   - 确保所有 View 组件的引用都已配置
   - 确保预制体引用正确

3. **检查资源**
   - 确保资源文件在正确位置
   - 确保 SpriteFrame 配置正确

## 四、GitHub 上传（必须完成）

### 步骤1：创建 GitHub 仓库

1. **登录 GitHub**
   - 访问 https://github.com
   - 登录你的账号

2. **创建新仓库**
   - 点击右上角 "+" -> "New repository"
   - 仓库名：`coco-poker`（或自定义）
   - 选择 Public 或 Private
   - **不要**勾选 "Initialize this repository with a README"
   - 点击 "Create repository"

### 步骤2：初始化 Git 仓库

1. **打开终端**
   - 在项目目录 `D:\coco_poker` 打开 PowerShell 或 CMD

2. **初始化 Git**
   ```bash
   cd D:\coco_poker
   git init
   ```

3. **添加远程仓库**
   ```bash
   git remote add origin https://github.com/你的用户名/coco-poker.git
   ```
   > 将"你的用户名"替换为你的 GitHub 用户名

### 步骤3：提交代码

1. **添加文件**
   ```bash
   git add .
   ```

2. **提交代码**
   ```bash
   git commit -m "初始提交：Cocos Creator 纸牌消除游戏"
   ```

3. **推送到 GitHub**
   ```bash
   git branch -M main
   git push -u origin main
   ```

4. **输入认证信息**
   - 如果提示输入用户名和密码
   - 用户名：你的 GitHub 用户名
   - 密码：使用 Personal Access Token（不是账号密码）

### 步骤4：验证上传

1. **刷新 GitHub 页面**
   - 访问你的仓库页面
   - 应该能看到所有文件

2. **检查文件**
   - 确认所有代码文件都已上传
   - 确认文档文件都已上传

## 五、制作演示视频（必须完成）

### 步骤1：选择录制工具

#### 方式1：Windows 内置工具（最简单）

1. **打开 Xbox Game Bar**
   - 按 `Win + G` 键
   - 如果没有，在 Microsoft Store 搜索 "Xbox Game Bar" 安装

2. **开始录制**
   - 点击录制按钮（或按 `Win + Alt + R`）
   - 开始操作游戏

3. **停止录制**
   - 再次按 `Win + Alt + R`
   - 视频保存在：`C:\Users\你的用户名\Videos\Captures\`

#### 方式2：OBS Studio（专业推荐）

1. **下载安装 OBS**
   - 访问 https://obsproject.com/
   - 下载 Windows 版本并安装

2. **配置 OBS**
   - 添加"显示器采集"源
   - 设置录制参数（参考"GitHub上传和演示视频指南.md"）

3. **开始录制**
   - 点击"开始录制"
   - 演示游戏功能
   - 点击"停止录制"

### 步骤2：录制内容

录制以下内容（建议 2-5 分钟）：

1. **开场（10秒）**
   - 显示游戏界面
   - 或显示游戏标题

2. **功能演示（2-3分钟）**
   - **卡牌匹配**（30秒）
     - 点击主牌区的牌
     - 展示匹配动画
     - 说明匹配规则
   
   - **翻牌功能**（30秒）
     - 展示无牌可匹配的情况
     - 点击备用牌堆翻牌
     - 展示翻牌动画
   
   - **回退功能**（30秒）
     - 执行几个操作
     - 点击回退按钮
     - 展示回退动画
     - 连续回退多次
   
   - **游戏胜利**（20秒）
     - 快速演示完整流程
     - 展示胜利界面

3. **结尾（10秒）**
   - 显示项目信息
   - GitHub 链接
   - 或简单的结束语

### 步骤3：编辑视频（可选）

1. **使用 Windows 视频编辑器**
   - 打开"视频编辑器"应用
   - 导入录制的视频
   - 可以添加标题、音乐等

2. **或使用其他工具**
   - OBS Studio（简单编辑）
   - 剪映（简单易用）
   - Adobe Premiere（专业）

### 步骤4：上传视频

#### 方式1：上传到 GitHub Releases

1. **创建 Release**
   - 在 GitHub 仓库页面
   - 点击 "Releases" -> "Create a new release"
   - 版本号：`v1.0.0`
   - 标题和描述

2. **上传视频**
   - 将视频文件拖拽到 "Attach binaries"
   - 选择视频文件（建议 MP4 格式）

3. **发布 Release**
   - 点击 "Publish release"

#### 方式2：上传到视频平台

1. **上传到 YouTube 或 Bilibili**
   - 上传视频
   - 获取视频链接

2. **在 README.md 中添加链接**
   ```markdown
   ## 演示视频
   
   [点击观看演示视频](视频链接)
   ```

## 八、操作环境要求

### 系统要求

- **操作系统**：Windows 10 或更高版本
- **处理器**：Intel Core i3 或同等性能（推荐 i5）
- **内存**：4GB RAM（推荐 8GB）
- **硬盘空间**：至少 5GB 可用空间
- **显卡**：支持 OpenGL 3.3 或更高版本

### 网络要求

- 下载软件时需要网络连接
- GitHub 上传时需要网络连接
- 视频上传时需要网络连接

## 九、常见问题解决

### Q1: Cocos Creator 无法启动

**解决方案**：
- 检查系统是否满足要求
- 尝试以管理员身份运行
- 重新安装 Cocos Creator

### Q2: TypeScript 编译错误

**解决方案**：
- 检查 Node.js 是否正确安装
- 检查 `tsconfig.json` 配置
- 尝试重新打开项目

### Q3: Git push 失败

**解决方案**：
- 检查网络连接
- 检查远程仓库地址
- 使用 Personal Access Token 而不是密码

### Q4: 视频录制失败

**解决方案**：
- 检查是否有足够的磁盘空间
- 尝试使用其他录制工具
- 检查系统权限设置
