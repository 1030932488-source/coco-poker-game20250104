# 资源准备指南

## 一、卡牌图片资源

### 1. 卡牌正面图片

需要准备 **52 张**卡牌正面图片，对应 13 个点数 x 4 种花色。

#### 命名规则

建议使用以下命名规则：
- 格式：`card_face_{点数}_{花色}.png`
- 点数：0-12（对应 ACE 到 KING）
- 花色：0-3（对应 CLUBS, DIAMONDS, HEARTS, SPADES）

#### 示例文件名

```
card_face_0_0.png  // ACE of CLUBS (A♣)
card_face_0_1.png  // ACE of DIAMONDS (A♦)
card_face_0_2.png  // ACE of HEARTS (A♥)
card_face_0_3.png  // ACE of SPADES (A♠)
card_face_1_0.png  // TWO of CLUBS (2♣)
...
card_face_12_0.png // KING of CLUBS (K♣)
card_face_12_1.png // KING of DIAMONDS (K♦)
card_face_12_2.png // KING of HEARTS (K♥)
card_face_12_3.png // KING of SPADES (K♠)
```

#### 图片规格建议

- **尺寸**：建议 200x300 像素（或按比例缩放）
- **格式**：PNG（支持透明背景）
- **背景**：透明或统一背景色
- **风格**：保持一致的设计风格

### 2. 卡牌背面图片

需要准备 **1 张**卡牌背面图片。

#### 命名规则

```
card_back.png
```

#### 图片规格建议

- **尺寸**：与正面图片相同（200x300 像素）
- **格式**：PNG
- **风格**：与正面图片风格统一

### 3. 资源目录结构

将图片资源放置在以下目录：

```
assets/
└── resources/
    └── cards/
        ├── faces/
        │   ├── card_face_0_0.png
        │   ├── card_face_0_1.png
        │   ├── ...
        │   └── card_face_12_3.png
        └── card_back.png
```

## 三、在 Cocos Creator 中配置资源

### 1. 导入图片资源

1. **导入图片**
   - 将图片文件复制到对应的资源目录
   - Cocos Creator 会自动检测并导入

2. **配置 SpriteFrame**
   - 选中图片资源
   - 在属性检查器中确认 SpriteFrame 已生成
   - 检查图片是否正确显示

### 2. 配置卡牌预制体

1. **创建卡牌节点**
   - 在场景中创建新节点 `Card`
   - 添加 `Sprite` 组件
   - 添加 `CardView` 组件

2. **配置 CardView 组件**
   - **cardSprite**：拖拽 Sprite 组件
   - **cardBackSpriteFrame**：拖拽卡牌背面图片的 SpriteFrame
   - **cardFaceSpriteFrames**：按顺序添加 52 张正面图片的 SpriteFrame
     - 顺序：ACE(0-3) -> TWO(4-7) -> ... -> KING(48-51)

3. **保存为预制体**
   - 将节点拖拽到 `assets/prefabs/` 目录
   - 命名为 `CardPrefab`

### 3. 配置关卡配置文件

1. **创建 JSON 配置文件**
   - 在 `assets/resources/configs/` 目录创建
   - 命名：`level1.json`, `level2.json` 等

2. **配置关卡数据**
   - 参考 `assets/resources/configs/level1.json`
   - 设置主牌区和备用牌堆的卡牌配置

- 后续再替换为正式资源

## 五、资源优化建议

### 1. 图片压缩

- 使用工具压缩 PNG 图片
- 在保证质量的前提下减小文件大小
- 推荐工具：TinyPNG、ImageOptim

### 2. 图集打包

- 将多张小图片打包成图集（Texture Atlas）
- 减少 DrawCall
- 提高渲染性能

### 3. 资源预加载

- 在游戏开始前预加载所有资源
- 避免游戏过程中卡顿
- 提供加载进度提示

## 七、常见问题

### Q1: 图片导入后显示不正确

**A**: 
- 检查图片格式是否为 PNG
- 检查 SpriteFrame 是否正确生成
- 检查图片尺寸是否合理

### Q2: 卡牌显示顺序不对

**A**:
- 检查 `cardFaceSpriteFrames` 数组的顺序
- 确保按照点数 x 花色的顺序排列
- 参考命名规则检查文件名

### Q3: 图片文件太大

**A**:
- 使用图片压缩工具
- 适当降低图片分辨率
- 考虑使用图集打包

### Q4: 找不到资源文件

**A**:
- 确保文件在 `assets/resources/` 目录下
- 检查文件路径是否正确
- 确认资源已正确导入到 Cocos Creator

## 八、快速开始（使用占位图片）

如果暂时没有正式资源，可以：

1. **创建简单的占位图片**
   - 使用纯色方块作为卡牌
   - 添加文字标识点数
   - 快速测试游戏逻辑

2. **使用代码生成**
   - 在 `CardView` 中使用代码绘制简单图形
   - 或使用 Cocos Creator 的 Label 显示点数

3. **后续替换**
   - 等正式资源准备好后
   - 直接替换图片文件即可
   - 无需修改代码

---

**提示**：资源准备是游戏开发的重要环节，建议在开始编码前就准备好基础资源，这样可以更好地测试和调试游戏功能。
