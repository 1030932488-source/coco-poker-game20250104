# 重要说明

## 一、关于 TypeScript 编译错误

### 说明

如果你在 VS Code 或其他编辑器中看到 TypeScript 编译错误（如"找不到模块'cc'"），**这是正常的**。

### 原因

1. **Cocos Creator 环境**
   - 这些错误只在外部编辑器中出现
   - Cocos Creator 内置了类型定义
   - 在 Cocos Creator 中运行不会有这些错误

2. **类型定义位置**
   - Cocos Creator 的类型定义在引擎内部
   - 外部编辑器无法访问这些定义
   - 不影响实际运行

### 解决方案

#### 方案1：在 Cocos Creator 中开发（推荐）
- 直接在 Cocos Creator 中编辑代码
- 使用 Cocos Creator 内置的代码编辑器
- 不会有类型错误提示

#### 方案2：忽略外部编辑器错误
- 这些错误不影响项目运行
- 可以在 Cocos Creator 中正常编译和运行
- 可以忽略外部编辑器的错误提示

#### 方案3：配置 VS Code（可选）
如果需要使用 VS Code，可以：
1. 安装 Cocos Creator 扩展（如果有）
2. 配置 TypeScript 路径映射
3. 或者直接忽略这些错误

### 验证项目是否正常

**在 Cocos Creator 中**：
1. 打开项目
2. 查看控制台（Console）
3. 如果没有编译错误，说明项目正常
4. 点击预览按钮，如果能正常运行，说明一切正常

1. **在 Cocos Creator 中打开项目**

## 五、常见问题

### Q1: 项目无法在 Cocos Creator 中打开

**解决方案**：
- 确保使用 Cocos Creator 3.8.0 或更高版本
- 检查项目路径是否正确
- 尝试重新打开项目

### Q2: 编译错误

**解决方案**：
- 检查 Node.js 是否正确安装
- 检查 `tsconfig.json` 配置
- 在 Cocos Creator 中查看控制台错误信息

### Q3: 资源加载失败

**解决方案**：
- 检查资源文件是否在正确位置
- 检查 `CardResService` 的路径配置
- 查看控制台的错误信息

### Q4: 功能不工作

**解决方案**：
- 检查组件引用是否正确配置
- 检查场景节点结构是否正确
- 查看控制台的错误信息
